# DocOrchestrator (doc-orchestrator)

DocOrchestrator — это config-first оркестратор обработки документов: конвейер для растрирования PDF, определения типа документа и извлечения полей данных с использованием локальных VLM, с усиленной валидацией, нормализацией и верификацией результатов в структурированный JSON. [memory:196][memory:7][memory:9]

## Зачем это нужно
Цель проекта — превратить “папку со сканами” в воспроизводимый, расширяемый и измеримый процесс извлечения данных: документ → тип → набор полей → проверенный результат. [memory:71][memory:2]  
Проект ориентирован на работу в закрытом контуре (local-first): модели, обработка и артефакты находятся локально, без обязательного облака. [memory:7][memory:11]

## Ключевые принципы
- **Конфигурация важнее кода**: типы документов, модели, папки, правила именования и этапы задаются через JSON и могут управляться будущим UI. [memory:1][memory:197]
- Многофазный пайплайн: быстрый роутинг (маленькая VLM) → точное извлечение полей (основная VLM) → постобработка. [memory:4][memory:7]
- Достоверность данных достигается не только моделью, но и детерминированными правилами: валидация форматов, нормализация и верификация (логика проверки/согласованности/доменных ограничений). [memory:9][memory:2]
- Каждая стадия атомарна и отключаема через конфигурацию (позже — через UI). [memory:196]

## Возможности (план проекта)
- Вход: PDF-файлы в папке (в т.ч. “один документ = несколько PDF в подпапке”). [memory:196]
- Растрирование страниц в несколько разрешений (например 100 DPI для роутинга и 300 DPI для извлечения). [memory:4]
- Определение типа документа локальной VLM и создание стартового JSON (минимум: `doc_type`). [memory:7][memory:9]
- Сортировка исходных документов по папкам типов + хранение артефактов обработки рядом (JSON, изображения, манифесты). [memory:196]
- Извлечение полей по типу документа в JSON по схеме, с правилами обязательности/форматов и последующей нормализацией. [memory:9][memory:2]
- Отчёты, логи, статистика ошибок/отказов и тестовый прогон для сравнения моделей/предобработок (позже). [memory:2]

## Конфигурация
Проект использует JSON-оркестратор: `root`-конфиг + папка с типами документов (`docTypes/`), где каждый тип описывается отдельным JSON-файлом и подхватывается автоматически при старте. [memory:1][memory:196]

Ожидаемая структура (может меняться по мере реализации):
- `config/root.json` — пути, включение этапов, модели, шаблоны именования.
- `config/docTypes/*.json` — описания типов документов (id, признаки, целевые поля, правила).
- `config/schema/*.schema.json` — JSON Schema для валидации конфигов (чтобы ловить ошибки на старте). [web:52][web:51]
